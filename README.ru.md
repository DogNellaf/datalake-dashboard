# DataLake — Платформа розничной аналитики

[English](README.md)

Django-приложение для загрузки, обработки и визуализации табличных данных о продажах. Файлы проходят через четырёхуровневый конвейер обработки и предоставляются через интерактивный конструктор графиков и интерфейс предварительного просмотра данных.

---

## Возможности

- **Управление пользователями** — регистрация и вход с правами доступа уровня staff
- **Загрузка CSV** — настраиваемый разделитель столбцов, автоматическое определение кодировки (UTF-8 / CP-1251)
- **Четырёхуровневый ETL-конвейер**
  - **Raw** — оригинальный файл без изменений
  - **Ingest** — CSV с нормализацией в UTF-8
  - **Table** — Parquet с автоматическим определением типов столбцов
  - **Distilled** — Parquet с удалёнными дублирующимися строками
- **Проверки качества данных** — валидация схемы, обнаружение дубликатов и пропусков, поиск специальных символов, проверка числового преобразования
- **Интерактивный конструктор графиков** — выбор столбцов перетаскиванием, типы графиков: точечный / линейный / столбчатый (на основе Plotly)
- **Предварительный просмотр данных** — первые 100 строк в виде HTML-таблицы
- **Статистика по столбцам** — количество значений, пропуски, уникальные значения, мин / макс / среднее / стандартное отклонение для числовых столбцов
- **Экспорт CSV** — потоковая выгрузка обработанного набора данных
- **Интеграция с Django admin** — загрузка датасетов и переход к графику / просмотру / экспорту в один клик

## Архитектура

```
Загрузка из браузера
        │
        ▼
┌───────────────┐
│   Raw Layer   │  Оригинальная копия файла
└───────┬───────┘
        ▼
┌────────────────┐
│ Ingest Layer   │  CSV в UTF-8, схема без изменений
└───────┬────────┘
        ▼
┌───────────────┐
│  Table Layer  │  Parquet, ASCII-имена столбцов, числовые типы
└───────┬───────┘
        ▼
┌──────────────────┐
│ Distilled Layer  │  Parquet, дублирующиеся строки удалены
└──────────────────┘
```

Все уровни хранятся в директории `media/` в подпапках с именем временной метки (`YYYYMMDDHHMMSS`), что позволяет работать с несколькими датасетами одновременно.

## Установка

**Требования:** Python 3.11+

```bash
# Клонировать репозиторий
git clone <url-репозитория>
cd app

# Создать и активировать виртуальное окружение
python -m venv .venv
source .venv/bin/activate      # Linux / macOS
.venv\Scripts\activate         # Windows

# Установить зависимости
pip install -r requirements.txt

# Применить миграции базы данных
python manage.py migrate

# Запустить сервер разработки
python manage.py runserver
```

Откройте [http://127.0.0.1:8000](http://127.0.0.1:8000) в браузере.

## Использование

1. **Зарегистрируйте** учётную запись по адресу `/register/`.
2. **Войдите** и перейдите в панель **Django admin** (`/admin/`).
3. Нажмите **Добавить** в разделе *Неструктурированные данные* для загрузки CSV-файла. Укажите разделитель столбцов, соответствующий вашему файлу (по умолчанию: `;`).
4. После успешной загрузки запись появится в списке с тремя кнопками действий:
   - **График** — открыть интерактивный конструктор графиков
   - **Предпросмотр** — просмотреть первые 100 строк
   - **Экспорт CSV** — скачать обработанный файл
5. В конструкторе графиков перетащите столбцы из левой панели в зоны **Ось X** и **Ось Y**, выберите тип графика и нажмите **Построить график**.

### Поддерживаемые форматы CSV

- Разделитель: любой одиночный символ (по умолчанию `;`)
- Кодировка: UTF-8, UTF-8-BOM или CP-1251 (определяется автоматически)
- Заголовки: обязательны в первой строке
- Десятичный разделитель: `.` или `,` (запятые конвертируются автоматически)
